Session Handling

Next

Session handling is used for the web server to know who the user is or what the user is doing in a website. Because HTTP is a stateless protocol, storing of information in some way is needed to identify one user in the application from the other. This is especially useful for e-commerce websites where information in their cart has to be persistent all throughout their visit in the site.

Session

A session is a way of storing information during a user’s interaction to be used all throughout the website. A session ends when the browser is closed, which is done by default, or when the session timeout is reached. On the browser, a unique identifier, called the session id, is stored when the user accesses the site. Every time a request is made, the session id is passed to the web server and would be paired to the database to retrieve data associated with the session.

Cookie

A cookie is a small textual information stored in the user’s browser that is associated with a URL. The cookie jar is where the client stores the cookie’s text which will serve as an identifier in a session. When a user first accesses the application, the server will send a cookie. A server sends cookie only once throughout a session. If a request is made, the client will check if there is a cookie that is associated with the URL being requested. If there exists a cookie, the cookie will be given to the server. If there is none, a cookie will be issued to the client. There are two types of cookies:

  • Session Cookies

    This kind of cookie is kept until the browser is closed.

  • Persistent Cookies

    These cookies are stored in the browser until a session timeout is reached or when destroyed at will.
