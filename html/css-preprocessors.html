CSS Preprocessors

Previous

Next

CSS Preprocessors are preprocessing languages which are used by web developers to build code in CSS faster. It uses special CSS files that could contain variables, functions, mixins and etc. These special files will then be converted into regular CSS files so that web browsers can recognize them.

CSS Preprocessors make your CSS code more organized and easier to maintain. Writing long CSS code will be less tedious since the use of a preprocessor will lessen the burden of coding. This increases productivity on the developer’s part and it will also save you a lot of time.

One of the popular CSS Preprocessors are SASS and LESS:

SASS

SASS stands for Syntactically Awesome Stylesheets. This is the most common preprocessor that is used today. It allows the usage of variables, nested rules, inline imports and etc. Before using SASS, you must have Ruby installed first.

Syntax

SASS has two syntax options: SCSS also known as (Sassy CSS) and SASS. SCSS has a more flexible syntax than SASS and is fully compliant with the CSS syntax. It uses brackets and semicolons. On the other hand, SASS syntax uses indentations instead of brackets and it also omits the semicolon. It is not fully compliant with CSS syntax but it is much quicker to write.

Nesting

SASS provides nesting that reduces the amount of code that one has to write. You can nest CSS selectors to have a visual hierarchy of your HTML. This increases readability and outlines identifiable selectors. The same goes for LESS but it uses an @ sign instead of the dollar sign.

LESS

LESS is a "dynamic stylesheet language". It runs inside Node and works on Javascript. Less is designed to resemble CSS, therefore the syntax of the two is identical except for a bunch of extra features.

Variables

Variables start with the (@) sign. Values are assigned with a colon (:).LESS allows one to specify variables in one place then reuse these variables anywhere in the style sheet.

Mixins

Mixins in less are like functions in programming languaged. They are used to group CSS instruction and handle the repetition of declarations. To declare a mixin in LESS, place a . symbol before the variable.
