General Header Fields

Previous

Next

General Headers provide information for both request and response messages. The following are general headers in HTTP.

  • Cache-control

    The Cache-control general header field is used for controlling cache. It also tells whether a resource needs to be cached or not.

  • Connection

    The Connection general header field controls whether a connection is persistent or not. It allows the sender to specify options that are desired for that particular connection.

  • Date

    The Date general header field shows the date and time at which the request was generated.

  • Pragma

    The Pragma general header field indicates that something is cacheable. It is used to include generic directives that might apply to any recipient along the request/response chain.

  • Trailer

    The Trailer general header field indicates that a header is present at the end of the data to supply metadata that might be dynamically generated while the message body is sent.

  • Transfer-encoding

    The Transfer-encoding general header field indicates what type of transformation has been applied to the message body to safely transfer it between the sender and the receiver.

  • Upgrade

    The Upgrade general header field is used for protocol negotiation. It allows the client to upgrade or change to a different protocol on the same connection.

  • Via

    The Via general header field indicates where the request passed through.

  • Warning

    The Warning general header field carries additional information about the status or transformation of a message error in the message.
