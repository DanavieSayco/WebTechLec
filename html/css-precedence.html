CSS Rule Precedence

Previous

Next

When CSS selectors target the same HTML elements and have the same CSS properties with other selectors, it is important to know which CSS rule takes precedence.

Style sheets can have 3 origins: user agent, user and the author.These are sorted according to origin and importance, from the one with the lowest priority to the highest priority:

  • User agent normal declaration
  • User normal declaration
  • Author normal declaration
  • Author important declaration
  • User important declaration
  • User agent important declaration

User Agent

User agents are applications that you use to view the website like browsers (Chrome, Firefox, Safari, etc.). They have their own default style sheet.

User

Users are the ones who view the website. They can override the styles of author by optionally setting their own stylesheets.

Author

Authors are the one who create the website and embeds all the styles of a website in CSS and links that stylesheet in the HTML of the website.

Normal vs. Important

An “!important” declaration will always take precedence over a normal declaration. Styles with normal declaration are styles that are defined normally. Important declarations override the normal declarations regardless of where they are on the page. User important declarations overrides author declarations since this is to give users with special requirements control over the presentational aspect of the site.

Specificity

Every selector has its place in the specificity hierarchy. When two selectors apply to the same element, the selector that is more specific will override the regular ones.

Hierarchy:

  • Inline styles

    • Inline styles always overwrite any styles since they are attached directly to the HTML element to be styled, therefore having higher specificity.

  • ID selectors/Number of ID selectors

    • IDs are identifiers for page elements. Since it can only be used once in a web page, it has the highest priority in selector precedence.

  • Number of class selectors, attribute selectors, and pseudo-classes

    • A class selector has a higher specificity than attribute, it is defined by a dot with a class name. The next one are attributes, followed by the pseudo-classes.

  • Number of type selectors & pseudo-elements

    • Pseudo-elements are counted as normal elements, therefore having a lower specificity than selectors.

Order

Lastly, If two declarations have the same properties, origin and specificity, the latter specified wins. Rules that are last defined in the document tree will take precedence over the ones that have been defined earlier.
