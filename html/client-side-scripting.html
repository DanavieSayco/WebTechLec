Client-Side Scripting

Previous

Next

Client-side scripts are executed by a browser. They provide changes to web pages after they arrive at a browser. The scripts manipulate and display the content returned from the server. The source code is passed from the server to a computer over the internet and runs directly in a browser. As a whole, client-side scripts are used for appearance and interaction.

These scripts interact with CSS files and the HTML code of a website. It selects HTML elements and then manipulates those HTML elements to provide users an interactive experience when viewing a website.

Since client-side scripts rely on the userâ€™s computer, a slow computer will result to a script running slowly as well. It may not also run if a browser does not understand the scripting language.

JavaScript

The most commonly used scripting for the client side is JavaScript. It was developed by Brendan Eich of Netscape Communications and was first released in the year 1995. It was introduced with the browser Netscape Navigator.

JavaScript is an open, cross-platform and object-oriented scripting language running inside a browser. It manipulates elements on a webpage by means of a Document Object Model (DOM).

The standard version is defined by the ECMA-262 Specification. It delivers a standardized, international programming language based on JavaScript. ECMAScript is a standardized version of JavaScript.

Some JavaScript Frameworks:

  • jQuery

    • A JavaScript object library streamlining the behavior of JavaScript across different browsers.

  • MooTools

    • An object-oriented JavaScript framework that allows one to extend the functionality of your code.

  • Angular JS

    • A framework that extends the syntax of HTML to define data bindings, templates and etc.

  • Bootstrap

    • A framework that provides UI components and facilitates responsive web and application development.

  • Prototype

    • Another framework that eases the development of dynamic web applications.
