CSS Combinators

Previous

Next

Combinators are used to select elements based on relationships with other elements.

Descendant

A descendant combinator is whitespace (ie. Space, tab, line feed, carriage return, form feed) that separates two sequence of simple selectors. It selects the specified descendants or children of an element.

div p {
    color: coral;                    
}

This is another paragraph.

This is a code.

Child

The child combinator (>) selects elements that are direct children of an element.

div > p {
    color: orange;                    
}

This is a paragraph.

This is another paragraph.

This is a code.

This is a paragraph that is a descendant of the div but is inside a span.

Sibling

Sibling elements must have the same parent to be considered as siblings.

Adjacent Sibling

The adjacent sibling combinator (+) selects elements that are the adjacent sibling of an element. This sibling must immediately follow the element.

div + p {
    color: blurywood;                    
}

This is a code.

This is a paragraph outside the div. It immediately follows the div element. Both the div element and this p element are children of the body element.

This is just another paragraph outside the div.

General Sibling

The general sibling combinatory (~) selects elements that precedes the element in the first sequence. 

div ~ p {
    color: bisque;                    
}

This is a paragraph.

This is another paragraph.

This is a code.

This is a paragraph outside the div.

This is a code outside the div.

This is another paragraph outside the div. The paragraphs outside the div element precede the div element. The two paragraphs and the div share the same parent, the body element.
