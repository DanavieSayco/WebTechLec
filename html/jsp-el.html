Expression Language

Previous

Next

JSP Expression Language (EL) makes it possible to easily access application data stored in JavaBeans components. JSP EL allows the creation of arithmetic and logical expressions. Within a JSP EL expression, integers, floating point numbers, strings, the built-in constants true and false for boolean values, and null can be used

Syntax:

${expression}

Operators

  • .

    • Access a bean property or Map entry

  • []

    • Access an array or List element

  • ()

    • Group a subexpression to change the evaluation order.

  • +

    • Addition

  • -

    • Subtraction or negation

  • *

    • Multiplication

  • / or div

    • Division

  • % or mod

    • Remainder

  • == or eq

    • Equality

  • != or ne

    • Inequality

  • < or lt

    • Less than

  • > or gt

    • Greater than

  • && or and

    • Logical AND

  • || or OR

    • Logical OR

  • ! or not

    • Unary Boolean complement

  • empty

    • Test for empty variable values

Implicit Objects

  • pageScope

    • Maps the given attribute name with the value set in the page scope.

  • requestScope

    • maps the given attribute name with the value set in the request scope

  • sessionScope

    • Fetches attributes from session scope, set by session object.

  • applicationScope

    • Maps the given attribute name with the value set in the application scope

  • param

    • Requests parameters as strings

  • paramValues

    • maps the request parameter to an array of values

  • header

    • Maps the request header name to a single value

  • headerValues

    • Used for fetching all the HTTP request headers.

  • cookie

    • Maps the given cookie name to the cookie value

  • initParam

    • Maps the initialization parameter.

  • pageContext

    • Provides access to many objects request, session etc.
