SQL Injection (SQLi)

SQL injection refers to injecting an SQL statement to bypass authentication or to modify or retrieve data from the database. The attacker will find for an input form that is included inside of an SQL query. This would only work though if the website simply included the user input in the query statement.

For example, the SQL query is as follows:

query = “SELECT id FROM users WHERE username = $username AND password = $password”

If the attacker puts password’ OR 1=1 on the password field, the query will be evaluated as true, even when other conditions are false. This is because an or is used and one of the conditions is true, which is 1=1. The attacker will then be logged in, most likely as the first account from the query list.

This can be prevented by using Prepared Statements in making a query where the values included in the query are determined by the user.

Cross-site Scripting (XSS)

Cross-site scripting is injecting a malicious script in a website. In order to run malicious JavaScript code in a victim’s browser, an attacker must first find a way to inject a payload into a web page that the victim visits. In order for an XSS attack to take place the vulnerable website needs to directly include user input in its pages. An attacker can then insert a string that will be used within the web page and treated as code by the victim’s browser.

If the website simply prints the user’s input as is, then a script could be run when the input looks like the following:

<script>doSomething();</script>

When the page loads, the script will be executed. Not only the <script> tag can be used in performing the attack. The <body>, <img>, <iframe>, <input>, <link>, <table>, <div>, and <object> tags can be used as well.

Cross-Site Request Forgery (CSRF)

CSRF attacks include a malicious exploit of a website in which a user will transmit malicious requests that the target website trusts without the user’s consent. In Cross-Site Scripting (XSS), the attacker exploits the trust a user has for a website, with CSRF on the other hand, the attacker exploits the trust a website has against a user’s browser.

For example, the attacker will give out a link to the user that would enable a script to run without the user’s knowledge. The link can be given as a plain link or as an image.

<img src="http://bank.example.com/withdraw?account=bob&amount=1000000&for=Fred">

Upon clicking on the link, the script will be executed. The following transfers 1 000 000 from Bob's account to Fred's.
