HTML History

Next

HTML 1.0 (1989 - 1994)

First version of HTML was released and it supported inline image and text controls. The disadvantages of HTML 1 are unable to use tables or frames, specify fonts, change page background or use forms. The web page may look similar to background and font used because of the limitations. The World Wide Web did not formally specify HTML 1.0 specification because they did exist when the HTML 1.0 was released. Lynx and Mosaic supported HTML 1.0.

HTML 2.0 (1995)

HTML 2.0 supported more browser and it was considerably improved. HTML 2.0 supported forms with a limited set of form elements such as text boxes and option button, change of page background and use of tables. Some browser maker makes their own features that are not part of actual HTML specification. The year 1995, W3C was established and HTML 3 was immediately superseded.

HTML 3.2 (January 1997)

HTML 3.2 supported creating tables, expanded the options for form elements, and it allowed to use complex mathematical operation in web pages. HTML 3.2 was created instead of HTML 3.0 because W3C did not immediately agreed on the next version of HTML 2.0. The specification of HTML 3.2 supported CSS but the browsers that time did not support it very well. The downside of HTML 3 is that the style is included directly in the webpage.

HTML 4.0(December 1997)

Developed under a code name Cougar. It became W3C Recommendation in 1997. In HTML 4, it became possible to change the appearance and look using the style sheets.

HTML 4.01 (1999)

HTML 4.01 had support for style sheets and scripting ability for multimedia elements. It also focused in separating style sheet and the actual content because of HTML 3 that included style in the actual content which is difficult to maintain. It had 3 flavors which are transitional, Strict and Frameset. Transitional are the depreciated elements and attributes. Strict does not have depreciated element and attributes. Frameset support frames and inline frames.

XHTML 1.0 (2000)

XHTML is combination of HTML and XML. XHTML meaning is Extensible Hypertext Markup Language. It takes after HTML but enforces stricter rules on it. XHTML is a reformulation of HTML in XML. It is considered as a separate language from HTML. XHTML is defined as a markup language that is rich in structural functionality but relies on the style sheet for presentation.

XHTML 1.1

XHTML 1.1 is also known as XHTML basic, geared toward mobile applications. Not much different from XHTML 1.0, the new thing about it is the introduction of modules which are the forms module, table module, and others.

XHTML 2.0

XHTML 2.0 is general purpose markup language with no backward compatibility for older versions.

HTML 5 (2014)

Developed by Web Hypertext Application Technology Working Group (WHATWG) consisting of Ian Hickson, Apple, and Mozilla. It was standardized in 2014. WHATWG wanted to develop HTML as a "Living Standard" which always updated and improved.
