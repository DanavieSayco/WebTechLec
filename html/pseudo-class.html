CSS Pseudo-class

Previous

Next

Pseudo-class is an implicit class that enables selection of elements based on information that is not a part of the structure of the webpage. It starts with a colon (:) then the name of the pseudo-class. Some of these are mutually exclusive while others can be applied simultaneously.

Syntax

selector:pseudo-class {
    property:value;
}

Dynamic Pseudo-classes

Dynamic pseudo-classes are lost or acquired by an element when a user interacts with the element. They do not appear in the document source or tree.

Link pseudo-classes

Link pseudo-classes are used specifically for links in the webpage. These are used to differentiate a visited link and a link that has not been visited yet. They are also mutually exclusive, which means that only one of them is true at a given time. A link cannot be visited and not visited at the same time. These are browser-specific so they may not work for other browsers.

  • :link
    The :link pseudo-class is used for links that have not yet been visited.
    a:link {
    color: orange;                    
}
     This is a link
  • :visited
    The :visited pseudo-class is used for links that have been visited.
    a:visited {
    color: pink;                    
}
     This is a link. Click me.

User Action Pseudo-class

User action pseudo-classes are used for the selection of an element a user is interacting with. These are not mutually exclusive and so, they can be used simultaneously in an element.

  • :hover
    The :hover pseudo-class is used when the userâ€™s pointer is hovering over an element. Some devices may not support this pseudo-class, like mobile phones and tablets.
    a:hover {
    color: red;                    
}
     Hover me
  • :active
    The :active pseudo-class is used to activate elements. The effect can be seen when the user presses the mouse button and will be gone once the mouse button is released.
    a:active {
    color: yellow;                    
}
     Click me
  • :focus
    The :focus pseudo-class is used for forms and for moving the focus to the element.
    input:focus {
    background-color: white;                    
}

Target Pseudo-class

URLs can refer to an element within the document, which is called a target element. These URLs have # followed by an anchor identifier. This pseudo-class is used to style those target elements.

div:target {
    display:block;

Language Pseudo-class

The language pseudo-class is used for targeting elements with a specific language attribute. It is most often a two-letter language code or two language codes combined. The language specified may not be an actual language but it must not be left as blank.

p:lang(fr) {
    color: darkblue;
    background-color: deepskyblue;
}

UI Element States Pseudo-classes

  • :enabled
    The :enabled pseudo-class is used for elements that are enabled. It is mostly used on form elements.
    input:enabled {
    background-color: yellowgreen;
}
  • :disabled
    The :disabled pseudo-class is used for elements that are disabled. It is also used on form elements.
    input:disabled {
    background-color: palevioletred;
}

  • :checked
    The :checked pseudo-class is used for selecting checked input or option elements. This can only be used for radio buttons and checkboxes.
    input:checked {
    height: 25px;
    width: 25px;
}

    Female
  • :indeterminate
    The :indeterminate pseudo-class is for radio buttons and checkboxes that have an indeterminate state. They are neither checked nor unchecked.

Negation

The :not() pseudo-class negates the logic of the selector. Only simple selectors can be used as an argument. Nesting negations is invalid. 

:not(p) {
    color: darkorange;
}
